# Fintech Sentiment Engine

Pulls Google Play reviews for BNPL and neobank apps, clusters them into semantic topics with BERTopic, and generates a PM-ready strategic brief via Claude — all surfaced in a Streamlit UI.

**Target apps:** Chime · Cash App · Affirm · Klarna vs. Chase (cobrand baseline)

## Project Phases

| Phase | Module | Status |
|-------|--------|--------|
| 1 — Data collection | `src/data_collection.py` | ✅ Complete |
| 2 — Topic clustering | `src/clustering.py` | ✅ Complete |
| 3 — Brief generation | `src/brief_generator.py` | ✅ Complete |
| 4 — Streamlit UI | `app/streamlit_app.py` | ✅ Complete |

## Results

### Topic clusters (BERTopic · 850 reviews · 9 topics)

| Topic | Theme | Docs | Avg ★ |
|------:|-------|-----:|------:|
| 0 | Card payments, refunds & merchant friction | 251 | 3.8 |
| 1 | App update & login failures | 96 | 1.8 |
| 3 | Account closures, dispute denials & credit score damage | 76 | 1.8 |
| 4 | App quality — mixed satisfaction | 69 | 4.1 |
| 5 | Chime product experience (SpotMe, early direct deposit) | 62 | 4.4 |
| 6 | BNPL pay-over-time value proposition | 59 | 4.5 |
| 7 | Banking fundamentals (fees, deposits, customer service) | 54 | 3.8 |
| 8 | Cash App — fraud, unauthorized charges & deactivation | 47 | 3.6 |
| 9 | Affirm — loan terms & interest rate friction | 42 | 3.1 |

94 generic-positive short reviews excluded from clustering as noise.

### Sample brief output (`claude-sonnet-4-6`)

> **The substitution threat is structural, not transactional.**
>
> Based on Google Play sentiment analysis across ~850 reviews from Chime, Cash App, Affirm, Klarna, and Chase, the data does not show users explicitly switching from cobrand credit cards to BNPL. What it reveals is something more foundational: two distinct failure modes in incumbent financial products, and a new generation of fintech tools capturing users precisely where those failures are most acute.
>
> The two sharpest pain clusters are defined by access failures and trust breakdowns, not product features. Topic 1 (app update & login failures, 96 reviews, 1.8★, Chase 45%) shows that Chase's biggest sentiment problem isn't rates or rewards — it's that users routinely can't get into the product after a forced update. Topic 3 (account closures, dispute denials & credit score damage, 76 reviews, 1.8★, Affirm 36%) captures a deeper trust erosion: unexpected shutdowns, BNPL utilization impacting credit scores, and dispute processes that favor merchants over consumers...
>
> *Full brief generated live from cluster data in the Streamlit UI.*

## Setup

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env  # add ANTHROPIC_API_KEY
```

## Run

**Notebook** (end-to-end walkthrough):
```bash
jupyter notebook notebooks/01_explore.ipynb
```

**Streamlit UI:**
```bash
streamlit run app/streamlit_app.py
```

The pipeline auto-loads from `data/processed/` if results exist, or re-runs BERTopic from scratch (~60s on Apple Silicon).

## Tech

- **BERTopic 0.16.4** — `all-MiniLM-L6-v2` embeddings, UMAP + HDBSCAN, semantic topic reduction
- **Anthropic SDK 0.113.0** — `claude-sonnet-4-6`, streaming, structured system prompt
- **Streamlit 1.41.1** — topic table with rating heatmap, live brief streaming
- **google-play-scraper** — 200 reviews/app, English, newest-first
